Deletes existing statistics entry(ies).

DELETE /airports/{airport_id}/statistics/

Deletes a single or multiple statistics entries. The entries are specified by the path parameter for the airport, the query parameter for the carrier and the optional parameter for the time. If no time is provided, all statistics corresponding to the airport and carrier are deleted.
airport_id

Code of the airport for the statistics.

 path string
c_code

Code of the carrier for the statistics.

 query string
month_year

Time label for the statistics (e.g. '2003/6').

 query string
204 No Content

Empty response. Signals the statistics are deleted.

Post a new statistic.

POST /airports/{airport_id}/statistics/

Each statistic is uniquely identifiable by the airport, carrier and time label. The statistic must contain all of the required fields. If the statistic has the same unique identifiers as an already existing statistic, it will not be stored (use PUT instead). The query parameters as well as path parameters are not important here!

application/json

Request body must be a proper statistics dictionary enclosed in square brackets.

Example:

[ { "airport": "ATL", "carrier": "AA", "statistics": { "minutes_del": { "late_aircraft": 1209, "weather": 1722, "carrier": 1367, "security": 139, "total": 8314, "nat_avi_sys": 3817 }, "num_del": { "weather": 28, "security": 2, "late_aircraft": 18, "nat_avi_sys": 105, "carrier": 34 }, "flights": { "cancelled": 5, "on_time": 561, "total": 2, "delayed": 186, "diverted": 0 } }, "time": "2003/7" } ]

object
c_code

Code of the carrier for the statistics.

 query string
airport_id

Code of the airport for the statistics.

 path string
c_code

Code of the carrier for the statistics.

 query string
month_year

Time label for the statistics (e.g. '2003/6').

 query string
200 OK

Empty response. Signals that the dictionary has been stored.
